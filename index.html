BLE TROUBLESHOOTING
If ALL DEVICES… still shows nothing: the phone is not seeing any advertiser from this receiver. On USB Serial (115200) you should see BLE advertising as 'WhiteFox Irrigation' after Boot complete. If that line never appears, fix power/USB, wrong sketch/port, or a boot crash before BLE. If you upgraded Arduino ESP32, try Tools → Bluetooth mode (Bluedroid vs NimBLE). Latest receiver_node.ino disables WiFi modem sleep and periodically refreshes advertising when disconnected.
WATERING RULES
1) Normal watering: Soil below moisture threshold, inside time window, and cooldown clear.
2) Emergency watering: Soil below emergency threshold, ignores time window. Cooldown applies except one bypass per boot (first emergency start while cooldown is active). Sending WATER ON restores that bypass for the next time it is needed.
3) Manual watering: WATER ON clears cooldown and starts immediately; full cooldown applies from pump stop.
4) Cooldown: Any new watering run (auto, emergency, or manual) clears the prior lockout timer on the controller.
5) Flow guard: After a prime period, if smoothed flow stays below the firmware minimum while the pump is on, the run aborts (tune constants in receiver_node.ino).
6) Dry bench test: Settings → Advanced → 15 FLOW CHECK ENABLED = OFF (0) disables that abort; timer stop still applies. Turn ON (1) for normal use; SAVE CONFIG to persist.
TIME WINDOW
Use 24-hour values.
Examples: 07 = 7am, 14 = 2pm.
For 11am to 2pm set:
05 WATER WINDOW START = 11
06 WATER WINDOW END = 14

KEY VALUES
04 MOISTURE THRESHOLD PERCENT: normal dry trigger
07 EMERGENCY THRESHOLD PERCENT: critical dry trigger (must be lower than #4)
01 WATERING DURATION: auto run time
02 WATERING COOLDOWN: lockout after run
03 MANUAL WATERING DURATION: WATER ON run time
RTC + LOG
Settings screen auto-refreshes sensor time when opened.
Use SYNC SENSOR TO PHONE TIME to set RTC quickly.
BLE LOG is live like terminal tail -f.
LONG-TERM HISTORY
The ESP32 · RUN LOG block loads from the controller over BLE (stats), including pump runs and the event log (flow guard, RTC/ESP-NOW, BLE errors). Phone-side charts use localStorage. EXPORT JSON archives this phone’s data. CLEAR STORED erases phone storage and, while BLE is connected, clears the ESP32 run/event history in flash (wfhist NVS) — same scope as a history-only factory reset.
