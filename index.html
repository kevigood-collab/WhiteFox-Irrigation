BLE TROUBLESHOOTING
Tap CONNECT to open the chooser: WhiteFox Irrigation opens the system Bluetooth picker (only real radios appear there — the browser cannot list a fake device). Choose Demo (no Bluetooth) to preview the UI without hardware. If connect shows nothing, the phone is not seeing the receiver advertiser. On USB Serial (115200) you should see BLE advertising as 'WhiteFox Irrigation' after Boot complete. If that line never appears, fix power/USB, wrong sketch/port, or a boot crash before BLE. If you upgraded Arduino ESP32, try Tools → Bluetooth mode (Bluedroid vs NimBLE). Latest receiver firmware (ESP32_Now_Protocol_Recieve.ino) disables WiFi modem sleep and periodically refreshes advertising when disconnected. Live status includes WF DEMO:0|1; when 1, the header shows an orange DEMO badge. On the ESP32, BLE commands demo on / demo off toggle synthetic showcase data.
MASTER SHUTOFF
Bottom of Settings → MASTER SHUTOFF / SYSTEM RESUME: tap to send master shutoff hold (pump off, all starts blocked) or master shutoff release. BLE disconnect clears the lock on the ESP32.
WATERING RULES
1) Normal watering: Soil below moisture threshold, inside time window, and cooldown clear.
2) Emergency watering: Soil below emergency threshold, ignores time window. Same cooldown as normal runs — no emergency start until cooldown has expired.
3) Manual watering: WATER ON clears cooldown and starts immediately; full cooldown applies from pump stop.
4) Cooldown: Any new watering run (auto, emergency, or manual) clears the prior lockout timer on the controller.
5) Flow guard: After a prime period, if smoothed flow stays below the firmware minimum while the pump is on, the run aborts (tune constants in ESP32_Now_Protocol_Recieve.ino).
6) Dry bench test: Settings → Advanced → 15 FLOW CHECK ENABLED = OFF (0) disables that abort; timer stop still applies. Turn ON (1) for normal use; SAVE CONFIG to persist.

LONG-TERM HISTORY
NVS history on the ESP32 holds litres + run counts, pump-off cumulative litres (leak estimate), plus Recent Runs and Event Logs. Opening History sends history; the ESP replies WF HT with totals (including OL= on current firmware), then WF RH/WF EH rows. CLEAR STORED sends clear history (reply OK history cleared).
